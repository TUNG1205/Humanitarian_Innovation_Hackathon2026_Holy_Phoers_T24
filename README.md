# Lami, Fiji — Climate Resilience Digital Twin

A digital twin dashboard for climate change adaptation in Lami, Fiji, built for the Humanitarian
Innovation Hackathon (Challenge B — digital twins for rural communities in the Pacific Islands).

Primary users are NGOs and local disaster management officers. It combines a live weather feed,
sea-level and impact projections, disaster history, an interactive evacuation/drainage/susceptibility
map, recovery protocols, AI-generated risk assessments, and offline-readable emergency guidance.

## Running the project

```
npm install
npm run dev
```

The Forecast tab's AI risk assessments need a backend function (see below) to work locally — without
it, that panel falls back to static baseline data automatically.

## Running the backend locally

The `api/risk-assessment.ts` serverless function calls Groq (Llama 3) and needs the [Vercel CLI](https://vercel.com/docs/cli):

```
npm i -g vercel
cp .env.example .env      # then fill in GROQ_API_KEY
vercel dev
```

`vercel dev` serves both the Vite frontend and the `/api` functions together, proxying requests so
`fetch("/api/risk-assessment")` from the browser works exactly as it will in production.

## Project structure

```
src/
  App.tsx                 Root shell — header, tab routing
  main.tsx                Entry point
  components/
    Header.tsx             Top nav: region, alert level, connectivity status, tabs
    shared/                Reusable UI atoms (StatCard, SectionTitle, RiskBadge)
    tabs/                  One component per dashboard tab
  data/                    Domain datasets (disasters, emergency contacts, climate normals, etc.)
  hooks/
    useWeather.ts           Live current conditions + 7-day forecast (Open-Meteo, no API key)
    useRiskAssessment.ts     Live AI risk narrative (calls api/risk-assessment.ts)
    useOnlineStatus.ts       Real browser online/offline detection
  lib/theme.ts             Shared design tokens (fonts, colors, radius)
  styles/                  Tailwind v4 + CSS custom properties theme
api/
  risk-assessment.ts       Vercel serverless function — the only place the Groq API key is used
```

## Data

- **Live**: current weather, 7-day forecast, and air quality via [Open-Meteo](https://open-meteo.com/),
  a free API with no key required. See `src/hooks/useWeather.ts`.
- **AI-generated**: the Forecast tab's risk outlook is generated by Llama 3 (via Groq) from live
  conditions — see `api/risk-assessment.ts`. Requires `GROQ_API_KEY` server-side; the frontend never
  sees it. Falls back to static baseline text (`src/data/riskFallback.ts`) if the backend is unavailable.
- **Curated**: disaster history, emergency contacts, and impact projections are sourced from public
  records (ReliefWeb, DFAT, Fiji Government/NDMO, UNDP SIDS Fiji country profile) — each dataset file
  in `src/data/` notes its source.
- No personal data is collected. Any future geolocation use would stay client-side only.

## Deploying

Deploy to Vercel (auto-detects the Vite frontend and the `/api` functions). Set `GROQ_API_KEY`
in the Vercel project's environment variables — never commit it.
